DDMusic API Documentation

基于当前 ddmusic-nextjs 系统自动梳理的第一版接口文档，覆盖公开音乐聚合接口、后台配置接口、认证接口，以及 README 与 HTML 文档的后台编辑能力。

公开访问入口

音乐聚合 API 对外入口为 /api，文档展示页入口为 /docs。

后台管理入口

管理控制台位于 /dashboard，可编辑 Provider、API 配置、README、HTML 文档和系统设置。

文档存储策略

README 直接落盘到 README.md，API 文档直接落盘到 doc/doc.html，然后镜像到数据库并可同步 GitHub。

项目概览

当前系统是一个基于 Next.js 16 的音乐 API 网关与后台管理平台。核心能力包含 GitHub OAuth 后台登录、Provider 与 API 配置管理、运行状态面板、README 在线编辑，以及 HTML 文档页在线编辑与发布。

公开音乐 API

对外主要通过 GET /api 使用查询参数驱动不同能力。当前真实实现的能力以 type 参数区分。

GET /api toplists

获取排行榜列表

按平台返回公开榜单列表。目前代码中支持 netease、qq、kuwo。

Queryprovider=tunehub, source=netease|qq|kuwo, type=toplists
Response统一返回 code、message、data
GET /api?provider=tunehub&source=netease&type=toplists
GET /api toplist

获取榜单歌曲

根据榜单 ID 返回榜单歌曲数据。当前支持 netease、qq、kuwo。

参数必填说明
source是平台标识：netease / qq / kuwo
type是固定为 toplist
id是榜单 ID
 
GET /api?provider=tunehub&source=qq&type=toplist&id=26
GET /api playlist

获取歌单详情

根据歌单 ID 返回歌单详情与歌曲列表。代码中对 kuwo 走公开服务，对其他平台走服务商聚合实现。

GET /api?provider=tunehub&source=netease&type=playlist&id=123456
GET /api search

搜索歌曲

当前真实代码支持平台歌曲搜索，默认结果数量 limit=20。实现中仅在指定平台创建搜索服务时生效。

GET /api?provider=tunehub&source=netease&type=search&keyword=周杰伦&limit=20
GET /api info

获取歌曲信息

根据歌曲 ID 获取歌曲基础信息。当前已经实现 info，而 url、pic、lrc、aggregateSearch 在公开入口中仍返回未实现状态。

GET /api?provider=tunehub&source=netease&type=info&id=2608813264
注意：type=url、type=pic、type=lrc、type=aggregateSearch、type=status、type=stats 当前代码中分别返回 404 或 501，不应在外部文档中描述为已上线。

后台数据接口

管理后台通过一组内部接口读写配置与文档，这些接口并不是公开业务 API，但它们决定了后台可编辑哪些内容。

GET POST Internal

/api/data/docs/readme

读取和保存 README.md。保存时顺序为：写本地文件、镜像到数据库、按配置提交到 GitHub 仓库。

GET  /api/data/docs/readme
POST /api/data/docs/readme
Body: { "content": "# Markdown" }
GET POST Internal

/api/data/docs/api

读取和保存 doc/doc.html。这是 /docs 页面实际渲染的数据源，不再使用 JSON 文档格式。

GET  /api/data/docs/api
POST /api/data/docs/api
Body: { "content": "<!DOCTYPE html>..." }
GET POST Internal

/api/data/api

读取和保存后台 API 配置。当前示例配置中存在一个 POST /v1/parse 解析接口，备注为“解析歌曲，额度是有次数限制的”。

{
  "apis": [
    {
      "name": "获取歌曲信息-tunehub",
      "path": "/v1/parse",
      "method": "POST"
    }
  ]
}
GET POST Internal

/api/sys

读取和保存系统配置。当前后台填写的 GitHub 仓库地址会作为 README 与 HTML 文档同步的目标仓库来源。

{
  "configuration": {
    "githubProjectPath": "https://github.com/yokeay/ddmusic-nextjs",
    "apiTimeout": 300000,
    "maxConcurrentRequests": 100
  }
}
GET POST Internal

认证与会话

后台登录流程包含 /api/auth/github/login、/api/auth/github/callback、/api/auth/session 和 /api/auth/logout。

GET  /api/auth/github/login
GET  /api/auth/github/callback
GET  /api/auth/session
POST /api/auth/logout

文档编辑链路

当前版本已把 README 和 API 文档统一成“文件可编辑、后台可编辑、GitHub 可同步”的链路，满足直接基于管理系统维护项目文档的需求。

文档后台编辑格式落盘文件访问地址
项目说明MarkdownREADME.md仓库首页 / 后台文档中心
API 文档HTMLdoc/doc.html/docs
后台保存后，系统会先写入本地 UTF-8 文件，再镜像到 PostgreSQL/缓存，并根据系统配置中的仓库地址与令牌把文档提交到 GitHub 指定分支。

错误与约束

400

参数缺失

公开音乐 API 在缺少 source、id、keyword 等关键参数时会返回 400。

404 / 501

功能未实现

部分接口类型在代码中仍未落地，会直接返回未实现。文档已按当前代码状态进行了降级说明。

安全说明

后台 API 配置中可能包含上游服务商密钥或 Header。为了避免将敏感信息直接暴露在公开文档，本页不会自动输出完整密钥值。公开展示时应仅描述参数结构与接入方式，不应贴出真实凭证。

当前版本说明

  • README 文档现在直接读写 README.md，可通过后台文档中心编辑。
  • API 文档现在直接读写 doc/doc.html，/docs 页面直接渲染该 HTML 文件。
  • 旧的 JSON 文档文件已移除，避免后台继续把 API 文档错误保存成 JSON。
  • GitHub 同步目标仓库可从后台系统设置读取，保存文档后会按配置同步到远程仓库。
