公开接口

下列能力直接对应 src/app/api/route.ts 中当前已落地的 switch 分支，是对外最核心的访问入口。

GET /api

获取歌曲信息

根据 provider、source 和 id 返回歌曲基础信息。

参数 必填 说明
type 是 固定为 info。
source 是 平台标识，目前支持 netease、qq、kuwo。
id 是 歌曲 ID。
provider 否 服务商标识。当前默认使用 tunehub。
默认值：tunehub

调用示例

GET /api?type=info&source=netease&id=2608813264&provider=tunehub

响应示例

{
  "code": 200,
  "message": "success",
  "data": {
    "name": "示例歌曲",
    "artist": "示例歌手",
    "album": "示例专辑",
    "id": "2608813264",
    "platform": "netease",
    "url": "/api?type=url&source=netease&id=2608813264",
    "pic": "/api?type=pic&source=netease&id=2608813264",
    "lrc": "/api?type=lrc&source=netease&id=2608813264"
  }
}

当前行为说明

  • 缺少 source 或 id 时会返回 400。
  • 成功时返回统一的 { code, message, data } 结构。
GET /api

搜索歌曲

按平台搜索歌曲，当前由平台适配器直接处理，provider 参数不会参与分发。

参数 必填 说明
type 是 固定为 search。
source 是 平台标识，目前支持 netease、qq、kuwo。
keyword 是 搜索关键词。
limit 否 返回数量上限。
默认值：20

调用示例

GET /api?type=search&source=qq&keyword=周杰伦&limit=20

响应示例

{
  "code": 200,
  "message": "success",
  "data": {
    "keyword": "周杰伦",
    "total": 2,
    "results": [
      {
        "id": "123456",
        "name": "夜曲",
        "artist": "周杰伦",
        "album": "十一月的萧邦",
        "platform": "qq",
        "url": "/api?type=url&source=qq&id=123456"
      }
    ]
  }
}

当前行为说明

  • 缺少 source 或 keyword 时会返回 400。
  • 不支持的平台会返回 400 Unsupported platform。
GET /api

获取歌单详情

根据 source 和 id 返回歌单详情。kuwo 平台会走公开服务，其他平台走已注册的音乐服务。

参数 必填 说明
type 是 固定为 playlist。
source 是 平台标识，目前支持 netease、qq、kuwo。
id 是 歌单 ID。
provider 否 服务商标识。当前默认使用 tunehub。
默认值：tunehub

调用示例

GET /api?type=playlist&source=kuwo&id=3136604274

响应示例

{
  "code": 200,
  "message": "success",
  "data": {
    "id": "3136604274",
    "name": "示例歌单",
    "description": "示例歌单描述",
    "cover": "https://example.com/cover.jpg",
    "songs": []
  }
}

当前行为说明

  • 缺少 source 或 id 时会返回 400。
  • 服务端异常时返回 500，并透出 error 字段。

GET / POST /api/data/docs/api

API 文档 HTML 管理

读取、保存或重新生成 doc/doc.html。

参数 必填 说明
content 否 POST 时提交新的完整 HTML 文档。
regenerate 否 POST 为 true 时按当前源码与配置重建文档。

调用示例

POST /api/data/docs/api
{ "regenerate": true }

当前行为说明

  • GET 会在文档缺失、为空、是旧版占位内容或旧版 JSON 文档时自动重建。
  • POST regenerate=true 会忽略传入 content，直接覆盖 doc/doc.html。
