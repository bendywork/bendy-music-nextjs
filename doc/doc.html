顶点音乐聚合API文档

基于 ddmusic-nextjs 项目的音乐聚合接口使用指南

1. 项目介绍

本项目通过对外暴露统一的接口，内部对接不同的音乐服务商，实现了音乐资源的获取和播放。按照新的维护方式，我们只使用服务商的解析接口，其他公共接口直接使用原厂商的API。

目前维护的音乐平台：

  • 网易云音乐
  • QQ音乐
  • 酷我音乐

2. API基础信息

2.1 基础URL

所有API请求都基于以下基础URL：

http://localhost:3000/api

2.2 通用参数

参数名 类型 是否必填 默认值 说明
provider string 可选 tunehub 服务商类型，目前仅支持 tunehub
source string 必填 无 音乐平台类型，网易云为 netease，QQ音乐为 qq，酷我音乐为 kuwo
type string 必填 无 请求类型，如 info、url、pic、lrc、search、playlist、toplists、toplist 等

2.3 接口类型说明

本项目的接口分为两种类型：

  • 公开接口：直接对接音乐平台官方API的接口，不属于任何服务商
  • 解析接口：使用服务商API进行解析的接口，如获取音乐文件链接、歌词等

3.5 获取歌曲信息

获取歌曲的基本信息，包括播放链接、专辑封面和歌词。

网易云音乐

解析接口

获取基本信息
GET /api?provider=tunehub&source=netease&type=info&id={歌曲ID}
获取音乐文件链接

GET /api?provider=tunehub&source=netease&type=pic&id={歌曲ID}
获取歌词

4. 错误处理

常见错误响应

  • 400 Bad Request：请求参数错误或缺失，如缺少必填参数。
  • 404 Not Found：请求的资源不存在，如歌曲ID不存在。
  • 500 Internal Server Error：服务器内部错误，如API调用失败。
  • 501 Not Implemented：请求的功能尚未实现。

4.1 错误响应格式

{
  "code": 400,
  "message": "Missing source or id parameter",
  "data": null
}

5. 示例代码

5.1 获取网易云排行榜列表

// 使用 fetch API 获取排行榜列表 async function getNeteaseToplists() { try { const response = await fetch('http://localhost:3000/api?provider=tunehub&source=netease&type=toplists'); const data = await response.json(); if (data.code === 200) { console.log('排行榜列表:', data.data.list); // 处理排行榜数据 } else { console.error('获取排行榜失败:', data.message); } } catch (error) { console.error('请求错误:', error); } } // 调用函数 getNeteaseToplists();

5.2 获取歌单详情

// 获取歌单详情 async function getPlaylistDetail(playlistId) { try { const response = await fetch(`http://localhost:3000/api?provider=tunehub&source=netease&type=playlist&id=${playlistId}`); const data = await response.json(); if (data.code === 200) { console.log('歌单名称:', data.data.name); console.log('歌单描述:', data.data.description); console.log('歌曲数量:', data.data.songs.length); console.log('歌曲列表:', data.data.songs); // 处理歌单数据 } else { console.error('获取歌单失败:', data.message); } } catch (error) { console.error('请求错误:', error); } } // 调用函数 getPlaylistDetail('123456789');

6. 注意事项

1. 本项目仅用于学习和研究目的，请勿用于商业用途。

2. 音乐平台API可能会有变动，请及时关注官方API文档的更新。

3. 由于音乐平台的API限制，部分功能可能需要登录才能使用。

4. 请合理使用API，避免频繁请求导致IP被封禁。

5. 当QQ音乐和酷我的公共接口提供后，会更新相应的文档。
